Website’s And Languages

My Experience Learning And Creating A Website

Introduction

In my time studying the materials needed in order to create a successful website, I initially went in only knowing one thing. This one thing happened to be that HTML was a primary tool needed in order to create websites, but thats about it. I couldn’t be farther from wrong. Not only do you need to use HTML, but you must also use two other languages called CSS and JavaScript. You don’t know what you don’t know, so when typing this I still have yet to discover the rest of what these languages can do, but this is my experience with learning them and using them to create a website.

Learning The Materials

When I first started, I didn’t know much and had no idea where to start. I looked up on google some primary ways people create websites, but kept on getting directed to places that offered a drag and drop service for creating. Now, in some cases this can work, but for what I was going for I wanted to be able to create these websites all on my own using the programming skills that I would soon learn. So it was back to google that I went. After a little bit more of searching, I found out that most website developers use three primary languages when creating their websites. (“Best Programming Languages for Web Development”) These languages were HTML, CSS, and JavaScript.

I didn’t know much about them based on definitions alone, so I had to find a way to learn each and get familiar with how they worked. I soon stumbled upon a youtube channel called Giraffe Acadamy, which happened to be my savior. The channel had everything that I needed. In an introductory video that they had created, they stated that HTML is what I should start with, and then move on to the other stuff. (“HTML Full Course - Build a Website Tutorial”) So after the video was done, I went to their playlists and found their HTML tutorial videos.

These videos had so much information that I needed that I had to dig out my old journal that I hadn’t used for a while. While watching these videos, I wrote down every subject from Basic Tags, all the way up to Meta Data. (“HTML Tutorial”) It was a lot, but eventually I figured it out. After I had watched his HTML series, I decided to move on to the next language he said to learn, which was CSS. He luckily also had a very informative playlist of videos that I went and watched. (“Introduction | CSS | Tutorial 1”) Although the information was time consuming to learn, I still managed to do what I did with the HTML videos and wrote down the information from each unite.

When nearing the end of the CSS tutorials, I realized that I was running out of time and that if I wanted to watch the JavaScript tutorials, which were next on the list, then I wouldn’t have enough time to write this or make the website. So in a rush, I remembered a website that the channel had talked about called w3schools.com. (“JavaScript Tutorial”) This website was a gold mine of information that clarified any questions I had and taught me the basics of the JavaScript that I was missing.

After learning the remaining language that I needed, I began work on the website. I had to think about what the home page would look like, what the other linked pages would look like, and so on. I used the websites that I had found to do this. (“CSS Tutorial”) Two of my classmates were even relying on me to create this website as they wanted to use it to display their written requirements. So after taking the time to think, I created a simple homepage and typed up my friends pages. This left my own, which as of typing this I will put this very text onto my page later. And that was it, so I thought. I still needed to figure out a way to make an interactive presentation, but that wouldn’t be too hard.

So, after all of this, what really did I learn? Well, let’s take a look.

HTML

HTML is the Skeleton. It is the backbone of the entire website. Without it, you wouldn’t have a website to begin with. HTML is the foundation of the other three languages that I learned as the others build of of what HTML creates. Using the videos and website, I first learned about basic tags. There were a dozen basic tags that I started with learning, but there are still hundreds of more that I can use when building a website.

For example, the <h1> tag makes words look like this!

While the <i> and <u> tag makes things look like this!

Later I learned that you must format your pages by using the <head>, <body>, and <footer> tags. But this was only just the structure of the website, and added no uniqueness to what I looked at. Later on I learned how to implement links, which were extremely useful by using the <a> tag. Surprising right? “Why not use a <link> tag?” Well the <link> tag is used for CSS, not HTML. Many tags like that will not be useful for HTML but instead are used to implement other languages like CSS and JS.

Then I learned the <img> tag, which makes things like, well, Images!

I then learned how to add videos and iFrames to my website, which are pretty self explanatory. Once I had practiced these things on my own, I moved on to more to the practical things that most people could use for work.

Things like the <li> and <tabel> tags, which make lists and tables.

Finally I learned how to create inputs and meta tags. Inputs are basically useless if you are not using another language like JavaScript or PHP. Meta tags are used to make sure the browser knows what type of language your using, how to display your website, and much, much more.

 

CSS

CSS is like the skin of your website. It provides the layer of style that gives your page some variety. CSS can either be bland or beautiful, depending on how you use it. When I learned about CSS, I basically did the same thing that I did when I learned HTML. I went to Giraffe Acadamies youtube channel and used w3schools to map my way through the mess of learning this language. Although it was hard, I found that CSS is my favorite out of the three because of how it make make your page look unique.

With CSS, I learned how to style text.

I could now make text look like this!

With the added bonus of being able to make text look colorful, I could also make box’s and add borders around make text to create sections in my website. With borders come padding and margins, which added space in between and around text. I learned about positions, allowing me to move objects in my website relative to each other or to the whole webpage.

With Opacity I could make words fade away.

With Shadows I could make text pop out.

With Border-Radius I could round edges.

With all this customization, I couldn’t leave out changing fonts, which was pretty easy when I learned the font-family attribute. I could also change the font size with, well, the font-size attribute. And I could change the style to italic by using the font-style attribute.

Later on I learned how to create an External Style Sheet, which helped me when making large amounts of CSS and allowing me to just link the ESS to my HTML files. I learned how to give elements classes and ID, which helped when identifying specific tags. I learned about CSS Selectors, which is too complicated to explain.  And finally I learned about flexbox’s, which I never ended up using. This is why CSS was my favorite, it was just so fun to be able to customize everything!

 

JavaScript

To end it off, JavaScript is like the Brain of your website. It adds a layer of interaction to your website and allows for greater usability. Without it, your website would look good, but wouldn’t hold on to users as it would lose their interests. Although I didn’t get to learn much about JavaScript as I ran out of time, I’m sure that you can guess what it does based off how I described HTML and CSS. JavaScript is one of the most used languages out there as it can be used for practically anything. If you wanted to make a videogame using JavaScript, you could, and I’m pretty sure someone in this class did just that. When this project is all said and done, I think that I’ll go back and re-visit JS just so that I can see its true potential.
